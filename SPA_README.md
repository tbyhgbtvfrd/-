# SPA (Single Page Application) - Документация

## Описание
Реализована функциональность SPA для загрузки страницы "Задания" без перезагрузки страницы.

## Как это работает

### Основные функции:
1. **Навигация на /tasks** - При нажатии на кнопку "Задания" загружается содержимое из `/talks/talks.html`
2. **URL адресная строка обновляется** - URL меняется с `/` на `/tasks`
3. **Возврат на главную** - При нажатии на кнопку "VPN" возвращается на главную страницу
4. **History API** - Поддерживаются кнопки назад/вперед браузера
5. **Сохранение состояния** - При загрузке страницы с URL `/tasks` автоматически загружается контент задания

### Файлы:
- `/vpnul.codes/index.html` - Главная страница (подключен скрипт `spa.js`)
- `/vpnul.codes/spa.js` - Основной скрипт маршрутизации
- `/talks/talks.html` - Страница с контентом задания

### Технологии:
- **Fetch API** - для загрузки содержимого
- **History API (pushState/popstate)** - для управления URL и навигацией
- **DOMParser** - для парсинга загруженного HTML

### Особенности реализации:
- Структура папки `/talks` не изменена
- Содержимое загружается динамически без перезагрузки страницы
- Активный таб в навигации обновляется при переходе между страницами
- Поддерживается история браузера (back/forward buttons)

## Использование

### Для пользователя:
1. Нажимаем на кнопку "Задания" в нижней навигации
2. Содержимое загружается и адресная строка изменяется на `/tasks`
3. Можно использовать кнопку "Назад" браузера для возврата на главную страницу
4. При прямом переходе на `/tasks` содержимое задания загружается автоматически

### Для разработчиков:
Если нужно добавить новые страницы или маршруты, отредактируйте файл `spa.js`:

```javascript
navigate(path) {
  if (path === '/tasks') {
    // текущая реализация
  } else if (path === '/new-page') {
    // добавьте новый маршрут
    this.showNewPage(true);
    history.pushState({ page: 'new-page' }, 'New Page', '/new-page');
  }
}
```
